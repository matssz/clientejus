# Validacao de Mercado - ClienteJus

## Hipotese Principal

Advogados autonomos e pequenos escritorios perdem tempo organizando documentos, pendencias e status de clientes que chegam pelo WhatsApp.

O ClienteJus deve resolver primeiro a bagunca operacional do atendimento juridico, nao substituir um ERP juridico completo.

## Sinais Publicos Encontrados

### 1. WhatsApp faz parte da rotina juridica

Noticias juridicas recentes mostram que o WhatsApp ja aparece como meio relevante em comunicacoes entre advogado, cliente e Judiciario, desde que exista prova clara da ciencia do destinatario.

Impacto para o produto:

- O MVP pode comecar com mensagens prontas via link `wa.me`.
- Nao precisa usar API oficial do WhatsApp no MVP.
- O produto deve ajudar o advogado a manter contexto e rastreabilidade basica da comunicacao.

### 2. Documentos de cliente sao uma dor real

Conteudos do mercado juridico tratam perda ou extravio de documentos como situacao delicada para escritorios e advogados, com risco operacional, reputacional e juridico.

Impacto para o produto:

- Upload e organizacao de documentos entram no MVP.
- Checklist documental por tipo de caso entra no MVP.
- O produto deve reduzir documento perdido, documento incompleto e retrabalho.

### 3. Sistemas juridicos grandes tem atrito percebido

Reclamacoes publicas sobre softwares juridicos frequentemente citam suporte, bugs, dificuldade de uso, preco, relatorios e cancelamento.

Impacto para o produto:

- ClienteJus nao deve competir como ERP completo.
- O diferencial deve ser simplicidade, preco baixo e foco em rotina pequena.
- O MVP deve ter poucas telas e fluxo obvio.

## Dores a Confirmar nas Entrevistas

Use as entrevistas para confirmar ou derrubar estas hipoteses:

1. O advogado perde informacao importante no WhatsApp?
2. O advogado esquece documentos pendentes?
3. O cliente pergunta status com frequencia?
4. O advogado sente falta de um checklist simples por tipo de caso?
5. O advogado pagaria R$ 49 por mes para organizar isso?

## Dores Prioritarias para o MVP

1. Cliente espalhado no WhatsApp.
2. Documento pendente sem controle.
3. Status do caso repetido manualmente.
4. Falta de visao rapida do que precisa de acao.

## Nao Construir Agora

- IA para peticoes.
- Integracao com tribunais.
- DataJud.
- Calculo de prazos.
- API oficial do WhatsApp.
- App mobile.
- Financeiro completo.
- Multiempresa avancado.

## Criterio de Avanco

So avancar para features alem de clientes, casos, documentos e checklists depois de:

- 3 entrevistas feitas.
- 1 advogado aceitar testar.
- 1 dor aparecer em pelo menos 2 entrevistas.
- 1 tela ou fluxo ser validado com um advogado real.
